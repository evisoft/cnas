# Project Kick-off Meeting

> Anchored to TOR ID(s): R2400 (Task 1.1, Milestone M1). This file is the
> canonical template for the formal kick-off held at contract signature.

## 1. Purpose

Establish the operating ground rules between Beneficiary (CNAS), Contractor,
and downstream stakeholders (MMPS, MF, ANSA, CNAM, CSP, banks). Confirm scope,
governance, communication channels, and acceptance criteria for Milestone M1.

## 2. Scope

- Single in-person session at CNAS HQ + remote bridge.
- Output: signed minutes, contact roster, decision log, action register.
- Pre-reads: TOR (`tor/TOR.md`), running register (`TODO.md`), iteration 1 PMP draft.

## 3. Agenda

1. Welcome and roster confirmation (Beneficiary + Contractor + MGov MCloud rep).
2. TOR scope walk-through — focus on §1 (obiective), §2 (functional reqs),
   §3 (non-functional reqs PSR/SEC/ARH), §4 (interoperability), §5 (data model).
3. Confirmation of the Iterative Waterfall delivery model (R2410) — 1-month
   iteration cadence, demo gates, status reporting cadence (R2413).
4. Communication channels: bi-weekly Steering Committee, weekly engineering
   sync, ad-hoc UX consultations (R2412), incident escalation channel.
5. Risk and assumption register — initial population.
6. Confirm Milestone M1 deliverables and acceptance criteria (PMP, SRS,
   tech-infra requirements, environment provisioning).
7. Decisions and action items (with owners + due dates).
8. Sign-off on minutes.

## 4. Cadence / Lifecycle

Single event. Re-run only if scope is materially renegotiated. Reconciliation
checkpoints in every bi-weekly status report (R2413).

## 5. Implementation map

- Governance artefacts live under `docs/pm/`.
- TOR reference: `tor/TOR.md`.
- Running register and per-iteration completion notes: `TODO.md`.
- Iterative delivery evidence: `.github/workflows/ci.yml`, weekly demo notes
  recorded against the periodic demo template (R2412 →
  [`periodic-demo-template.md`](periodic-demo-template.md)).
